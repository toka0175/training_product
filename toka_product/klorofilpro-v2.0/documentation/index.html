Prerequisites

Before you started, make sure you have installed Node.js and npm.

NOTE: You don't need these steps if you plan to use HTML-ready version at theme-dist/ directory

To check if you have Node.js installed, run this command in your terminal:

$ node -v

To check if you have npm installed, run this command in your terminal:

$ npm -v

If you don't have Node.js installed, please download and install it from Node.js. Note that installing Node.js also install npm.

Running Project on Local Development Server

In terminal, go to app root directory and run the following command to install dependencies

$ cd theme
$ npm install

It may takes some times. Once finished run command

$ gulp

After the process is finished, notice that a new directory dist/ is created. In this stage, you can get compiled theme HTML along with the assets, CSS and JS files on that directory.

For development purposes, you should watch files changes with the following command:

$ gulp watch

It should automatically open the site at http://localhost:3000 and the page will reload if you make edits on src/. You can either work based on dist/ directory or ideally you make edit at src/ and get the final result at dist/ directory. src/ directory provides separation of layout structure between header, main content and footer for easier development.

Layout Structure


Several main HTML elements you need to be familiar with:

  • #wrapper main container of the template
  • navbar navbar-expand fixed-top top navigation, fixed-top by default
  • #sidebar-nav left navigation menu
  • .main main container for the content
  • footer main container for footer

CSS and Javascript

You can find template CSS at assets/css/main.css and all skin files at assets/css/skins/ directory. If you use Sass, you will find all the .scss files at sass/ directory, with main file main.scss and skin Sass file at sass/skins/ directory. CSS files for 3rd-party plugins are located at each plugin directory at assets/vendor/.

Javascript file needed in all pages are:

  • assets/vendor/jquery/jquery.min.js
  • assets/vendor/bootstrap/js/bootstrap.min.js
  • assets/vendor/pace/pace.min.js"
  • assets/scripts/klorofilpro-common.js

Klorofil Pro uses many 3rd-party plugins, with its javascripts located at each plugin directory at assets/vendor/. A page might need plugins with its data, configuration and initialization scripts. You will find scripts needed on the bottom of each page with specific plugins.
